# Recap Cluster Architecture

Damit du alle Konzepte der Cluster API verstehst, musst du zuerst die Cluster Architecture kennen.

Ein Kubernetes Cluster besteht aus einer Control-Plane und mehreren Workers.
Jedes Cluster braucht mindestens eine Worker-`Node` damit `Pods` auf dem Cluster erzeugt und (auf der `Node`) ausgeführt werden können.
